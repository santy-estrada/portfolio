Differential Drive Robot

Remote and Autonomous Mobile Robot with Sensor Fusion, PID Velocity Control, and ROS2 Integration

Project Details

Completed: 2025

This project consisted in the design, construction, coding, and testing of a fully functional differential drive mobile robot capable of operating in both manual (remote) and autonomous modes.

The robot was developed for a competitive academic challenge where each team needed to complete a two-lap square circuit in the shortest time possible. My team achieved first place with a 30-second lead, demonstrating the effectiveness of the robot's control system, sensor fusion, and ROS2 architecture.

Mechanical Design

The robot used a differential drive configuration consisting of:

  • Two DC motors with encoders driving the left and right wheels.
  • One caster wheel for passive stabilization.
  • Front-facing ultrasonic sensor for obstacle avoidance.
  • IMU integrated for orientation measurement and fusion with encoder odometry.

The chassis was designed for modularity and robustness, enabling quick motor replacement and easy access to electronics.



Sensing and Safety

  • Ultrasonic sensor: When an object was detected within 10 cm, the robot force-stopped the motors and ignored joystick input.
  • IMU + Encoders: Implemented sensor fusion for improved heading estimation during autonomous navigation.

These systems ensured safety in manual mode and accuracy in autonomous operation.

Low-Level Control

The low-level control system implemented a discrete PID controller to regulate motor velocity.

Development steps included:

  1. Non-parametric system identification of the motor dynamics.
  2. Ziegler–Nichols tuning to obtain an initial PID approximation.
  3. Simulink simulations with disturbances for robustness testing.
  4. Final deployment on a Raspberry Pi Pico running micro-ROS.

The final PID loop executed every 100 ms, balancing real-time performance with available processing resources.

Electronics & Embedded System

The embedded system architecture used:

  • Raspberry Pi Pico running micro-ROS for motor control and sensor processing.
  • Raspberry Pi 2 as the onboard computer handling ROS2 communication.
  • Custom wiring harness for clean integration of sensors and motor drivers.

The Pico transmitted odometry and received velocity commands through micro-ROS topics, while the Raspberry Pi 2 acted as a ROS2 bridge with the remote workstation.

High-Level Control and Autonomy

The high-level control system supported two modes:

  • Manual Mode — A ROS2 joystick node interpreted controller commands and sent motor references through the network.
  • Autonomous Mode — A set of ROS2 nodes implemented trajectory following using the pure pursuit algorithm.

Real-time visualization was achieved using RViz, showing the robot's path, target trajectory, and live sensor feedback.

Testing, Analysis, and Competition

Extensive testing was performed in controlled environments to validate velocity control, odometry accuracy, sensor fusion, and trajectory following.

Data from experiments was exported and analyzed in MATLAB to adjust parameters, validate simulations, and refine controller response.

In the final competition, the robot completed two laps of the square circuit significantly faster than other teams, securing first place.
