N-Queens

Use of recursive algorithms to solve the N-Queens problem

Project Details

Completed: October 2023

The n-queens problem is a classic combinatorial challenge where the objective is to place n queens on an n x n chessboard such that no two queens can attack each other. This means that no two queens should share the same row, column, or diagonal. For a board size of 2 or 3 queens, the problem has no solutions due to insufficient space to place the queens without conflict. However, for a 4x4 board, there are 2 distinct solutions. As the board size increases, the number of possible solutions grows exponentially. For an 8x8 board, there are 92 distinct solutions. The time complexity of solving the n-queens problem using a backtracking algorithm is O(n!), which reflects the factorial growth in the number of permutations that need to be explored. The size of the solution space also expands significantly with larger n, making the problem increasingly challenging to solve efficiently as n grows.

The first step of my project involved designing an algorithm in Java using recursion to find and store all possible solutions to the n-queens problem. Initially, I used a three-dimensional array to capture each board configuration as a potential solution. This approach allowed me to represent the entire board setup for each configuration in a structured manner. Subsequently, I optimized the storage of solutions by transferring each complete board configuration into a two-dimensional array, where each board was saved as a vector. This change streamlined the storage process, making it more efficient to handle and retrieve multiple solutions by leveraging the two-dimensional array's capability to manage all stored configurations compactly.

The final product of the project featured a graphical user interface (GUI developed using JFrame) that visually represented the solutions to the n-queens problem. The interface displayed the entire chessboard, with each cell marked with a 'Q' where a queen was placed. Users could navigate through all possible solutions using the interface, with a "Next" button that allowed them to cycle through the solutions sequentially. If the user reached the last solution and selected "Next," the interface looped back to the beginning, providing a continuous viewing experience. This GUI made it easy to visualize and interact with the different board configurations, enhancing the overall usability and understanding of the problem's solutions.
